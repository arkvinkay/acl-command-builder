Pengaturan

Generator ini membangun perintah berdasarkan opsi yang dipilih. Untuk skenario sensitif, uji terlebih dahulu di folder uji sebelum diterapkan ke path produksi.

Path akan di-quote otomatis jika mengandung spasi.
Membantu preset Linux “Otomatis” (folder=755, file=644).
Terapkan ke seluruh isi folder
Windows: /t · Linux: -R atau find -type f/d sesuai opsi.
Diperlukan untuk Grant/Deny/Remove/SetOwner.
Untuk “hanya turunan”, umumnya gunakan (IO).
Digunakan saat aksi = Atur inheritance.
Hak akses standar
Jika memilih hak akses lanjutan, output akan memakai format daftar (mis. (WDAC,WO,DC)).
Hak akses lanjutan (granular)
Gunakan ini jika membutuhkan kontrol lebih spesifik dibanding hak akses standar.
Catatan: Flag inheritance (OI)(CI) mengatur pewarisan ke objek turunan, sedangkan /t menerapkan perubahan ke objek yang sudah ada di bawahnya.
Mode rekursif menggunakan /s /d.
Workflow (Windows) — opsi lanjutan
Tambahkan takeown sebelum aksi utama
Membantu saat akses/owner bermasalah.
takeown /a
Owner menjadi grup Administrators.
Tambahkan icacls /reset sebelum aksi utama
Reset ACL terlebih dahulu (opsional).
icacls /c
Lanjutkan proses meski ada error pada beberapa item.
Aktif jika Mode = Expert.
Tambahkan prefix sudo
Nonaktifkan jika tidak diperlukan.
Direkomendasikan jika kamu ingin folder 755 dan file 644 secara terpisah.
